# AGENTS.md — tasty-nextjs-example

## Project Overview

Marketing site and interactive playground for `@tenphi/tasty`. Built with Next.js (App Router), statically exported. Hosted at [https://tasty.style](https://tasty.style).

Repository: [https://github.com/tenphi/tasty.style](https://github.com/tenphi/tasty.style)

## Quick Reference


| Command               | Purpose                                        |
| --------------------- | ---------------------------------------------- |
| `pnpm dev`            | Start dev server (runs `build:examples` first) |
| `pnpm build`          | Production build + static export               |
| `pnpm build:examples` | Regenerate playground examples bundle          |
| `pnpm lint`           | Lint source files                              |
| `pnpm lint:fix`       | Lint and auto-fix                              |
| `pnpm format`         | Format with Prettier                           |
| `pnpm format:check`   | Check formatting                               |
| `pnpm hygiene`        | Run lint + format check                        |
| `pnpm hygiene:fix`    | Auto-fix lint + format                         |


## Stack

- **Framework**: Next.js 16 (App Router, static export)
- **Styling**: `@tenphi/tasty` (runtime + zero-runtime via Babel plugin)
- **Color system**: `@tenphi/glaze` (OKHSL color themes)
- **Playground**: WebContainers (in-browser Node.js) + CodeMirror
- **Search**: Pagefind (post-build indexing)
- **Lint**: ESLint 10 + typescript-eslint + prettier + `@tenphi/eslint-plugin-tasty`
- **Format**: Prettier (single quotes, semicolons, trailing commas)
- **Package manager**: pnpm 10

## Project Structure

```
app/
  layout.tsx          Root layout
  page.tsx            Landing page
  theme.ts            Glaze color theme definitions
  states.ts           Custom state aliases
  global-styles.ts    Global tastyStatic() styles
  tasty-config.ts     Runtime configure() (states only)
  tasty-zero.config.ts  Full config for zero-runtime Babel plugin (typography, tokens)
  tasty-registry.tsx  SSR style registry wrapper
  fonts.ts            Font definitions (Onest, JetBrains Mono)

  components/         Shared site components (header, footer, showcases)
  ui/                 Reusable UI primitives (Button, Badge, etc.)
  docs/               Documentation pages (MDX-based)
  playground/         Interactive playground
    examples/         Source example files (.tsx) + config files
    lib/              Playground runtime (examples bundle, CodeMirror, WebContainer)
  lib/                Site utilities (Shiki, MDX helpers)
```

## Generated Files

`**app/playground/lib/examples.ts**` is generated by `scripts/build-examples.mjs`. **Do not edit it manually.** It bundles all `.tsx` files from `app/playground/examples/` plus `default-config.ts`, `default-global.ts`, and `icons.tsx` into string constants for the playground. To update it:

1. Edit the source files in `app/playground/examples/`
2. Run `pnpm build:examples`

The file is also regenerated automatically by `predev` and `prebuild` hooks.

## Configuration Files


| File                                        | Purpose                                                                           |
| ------------------------------------------- | --------------------------------------------------------------------------------- |
| `tasty.config.ts` (root)                    | ESLint plugin validation config (tokens, states, presets lists)                   |
| `app/tasty-zero.config.ts`                  | Full Tasty config for zero-runtime extraction (typography presets, design tokens) |
| `app/tasty-config.ts`                       | Runtime `configure()` call (states only, loaded client-side)                      |
| `app/playground/examples/default-config.ts` | Default config injected into playground examples                                  |
| `app/playground/examples/tasty.config.ts`   | ESLint validation config for playground example files                             |
